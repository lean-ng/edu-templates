Template Expressions

Der Ausdruck (Expression) der bei Property-Bindings und Interpolationen verwendet wird, ist kein JavaScript - man kann vielleicht von einem Subset reden.

Wichtige Merkmale/Unterschiede sind:
  • Kontext: Der Expression-Kontext ist die Komponenten-Instanz. Dementsprechend ist das this-Keyword gebunden auf die Komponenten-Instanz: {{ this.message }}
  • Globales Objekt: Der globale Kontext existiert nicht, das globale Window-Objekt ist also: {{ window === undefined ? 'undefined' : '' }}. Damit ist jegliche Benutzung globaler Variablen/Methoden/Klassen nicht möglich.
  • Seiteneffekte in Ausdrücken sind nicht erlaubt. Während also eine Zuweisung in JavaScript technisch gesehen ein Ausdruck ist, ist die Verwendung in Template-Expressions untersagt. Antwort: {{ answer }}
  • Pipe-Operator und Pipes: Zu den Syntax-Erweiterungen gehört die Neudefinition des |-Operators. Er dient zur Verknüpfung von Ausdrücken mit Pipes: {{ message | uppercase }}.

    Zu den vordefinierten Pipes gehören

    • uppercase, lowercase, titlecase
    • date
    • currency, decimal, percent
    • async
    • slice, keyvalue
    • json
    • i18nPlural, i18nSelect
  • Safe-Navigation Operator: dieser ist mittlerweile auch in JavaScript angekommen. Wird oft eingesetzt um Properties von Objekten zu rendern, die erst später definiert sind. Beispiel: {{ later?.msg }}
  • Non-Null-Assertion: Im Zusammenhang mit der TypeScript-Compilation und eingeschalteten Strict-Null-Checks eingesetzt. Fordert auf nullable-Properties einen not-null Wert. Beispiel: {{ nullable.prop! }}
  • Methoden-Aufrufe: Natürlich können auch Methoden-Aufrufe in Template-Expressions benutzt werden. Dies sollte allerdings nicht geschehen für Instanz-Methoden der Komponente auf Grund des Performance-Impacts. Message: {{ message.substr(0,5) }}
